React Is An Abstraction Over Vanilla JavaScript

Before React, developers manually managed State, UI, and DOM and had to keep them synchronized.

State → Developer → Manual DOM Updates → UI

React was created to take this responsibility away from developers.

React Manages

  • UI synchronization
  • State synchronization
  • DOM updates
  • Re-rendering
Important: React is NOT replacing JavaScript. React is built ON TOP OF JavaScript. React is an abstraction layer over Vanilla JavaScript.

Browser Does Not Understand React

Browser Understands

  • HTML
  • CSS
  • JavaScript

Browser Does NOT Understand

  • JSX
  • Components
  • Virtual DOM
  • Hooks
  • React-specific syntax

Why Do We Use React Then?

React Solves

  • Reusability
  • Component architecture
  • State management
  • UI synchronization
  • Rendering management

JSX Transformation

JSX

<h1>Hello World</h1>

Transforms To

React.createElement( "h1", null, "Hello World" );

Arguments

Argument 1: "h1" - Element type
Argument 2: null - Props object
Argument 3: "Hello World" - Children

Complex JSX Example

JSX

<div> <h1>Hello</h1> <button>Click Me</button> </div>

Transforms To

React.createElement("div", null, React.createElement("h1", null, "Hello"), React.createElement("button", null, "Click Me") );

What Does React.createElement Return?

Important: React.createElement does NOT create a real DOM node. It returns a plain JavaScript object.
{ type: "div", props: { children: [ { type: "h1", props: { children: "Hello" } }, { type: "button", props: { children: "Click Me" } } ] } }

This Object Is The Virtual DOM

Definition: Virtual DOM is a JavaScript object representation of the UI. It is NOT the real DOM.

What Happens During Re-render?

State Change → App Executes Again → New Virtual DOM → Comparison → DOM Updates
Note: React compares Old Virtual DOM vs New Virtual DOM. This is called Reconciliation.

Reconciliation

Definition

The process of comparing the previous Virtual DOM tree with the new Virtual DOM tree.

Old Virtual Tree + New Virtual Tree → Compare → Find Differences → Minimal DOM Updates

Internally React Uses Vanilla JavaScript

Important: React is simply managing DOM operations for us.
document.createElement() appendChild() removeChild() replaceChild() setAttribute() addEventListener()
Developer → React → ReactDOM → Vanilla JS DOM APIs → Browser

React Is An Abstraction Over Vanilla JavaScript

Before React, developers manually managed State, UI, and DOM and had to keep them synchronized.

State ↓ Developer ↓ Manual DOM Updates ↓ UI

React was created to take this responsibility away from developers.

React Manages

  • UI synchronization
  • State synchronization
  • DOM updates
  • Re-rendering
Important: React is NOT replacing JavaScript. React is built ON TOP OF JavaScript. React is an abstraction layer over Vanilla JavaScript.

Browser Does Not Understand React

Browser Understands

  • HTML
  • CSS
  • JavaScript

Browser Does NOT Understand

  • JSX
  • Components
  • Virtual DOM
  • Hooks
  • React-specific syntax
// This JSX cannot run in browser function App() { return <h1>Hello World</h1>; } // How can the browser execute this? // It cannot. Browsers don't understand JSX.

Why Do We Use React Then?

React Solves

  • Reusability
  • Component architecture
  • State management
  • UI synchronization
  • Rendering management

Instead of verbose vanilla JS, we write cleaner JSX:

Vanilla JS

const h1 = document.createElement("h1"); h1.textContent = "Hello World"; root.appendChild(h1);

React JSX

<h1>Hello World</h1>

JSX Is Not HTML

Many beginners think JSX is HTML. This is incorrect.

JSX is syntax that LOOKS like HTML. Internally it becomes JavaScript.

JSX Transformation

JSX

<h1>Hello World</h1>

Transforms To

React.createElement( "h1", null, "Hello World" );

Arguments Explained

Argument 1: "h1" - Element type
Argument 2: null - Props object
Argument 3: "Hello World" - Children
React.createElement( type, props, children )

Complex JSX Example

JSX

<div> <h1>Hello</h1> <button>Click Me</button> </div>

Transforms To

React.createElement( "div", null, React.createElement("h1", null, "Hello"), React.createElement("button", null, "Click Me") );
createElement("div") ├── createElement("h1") │ └── "Hello" └── createElement("button") └── "Click Me"

What Does React.createElement Return?

Important: React.createElement does NOT create a real DOM node. No browser element is created yet. Instead it returns a plain JavaScript object.
{ type: "div", props: { children: [ { type: "h1", props: { children: "Hello" } }, { type: "button", props: { children: "Click Me" } } ] } }

This Object Is The Virtual DOM

Definition: Virtual DOM is a JavaScript object representation of the UI. It is NOT the real DOM. It is only a description of what the UI should look like.

What Happens During Re-render?

setCount(count + 1);
Old State ↓ Old Virtual DOM State Change ↓ App Executes Again ↓ New Virtual DOM ↓ Comparison ↓ DOM Updates
Note: React does not compare real DOM first. React compares Old Virtual DOM vs New Virtual DOM. This process is called Reconciliation.

Reconciliation

Definition

The process of comparing the previous Virtual DOM tree with the new Virtual DOM tree.

Old Virtual Tree ↓ Compare ↑ New Virtual Tree ↓ Find Differences ↓ Minimal DOM Updates

React determines exactly what changed. Then ReactDOM performs only the necessary DOM operations.

Internally React Still Uses Vanilla JavaScript

Important: Many students think React eliminates DOM APIs. It does not. React is simply managing those operations for us.
// ReactDOM internally uses: document.createElement() appendChild() removeChild() replaceChild() setAttribute() addEventListener()
Developer ↓ React ↓ ReactDOM ↓ Vanilla JS DOM APIs ↓ Browser

React Is Two Cooperating Systems

  1. React Core: Component -> JSX -> Virtual DOM
  2. ReactDOM: Virtual DOM -> Reconciliation -> Real DOM

React helps developers describe UI. ReactDOM converts that description into actual browser elements using Vanilla JavaScript APIs internally.

Browser Rendering Pipeline

When a browser receives HTML, CSS, and JavaScript, it goes through a series of steps to convert code into pixels on the screen. This process is called the Browser Rendering Pipeline.

Step 1: HTML Parsing -> DOM

HTML -> DOM

The browser starts reading HTML from top to bottom. During parsing, it creates a tree-like structure called the Document Object Model (DOM). The DOM represents every HTML element as a JavaScript object that can be accessed and modified through the document object.

<!-- HTML Structure --> <body> <div id="app"> <h1>Hello World</h1> </div> </body> // DOM Structure Document └── html └── body └── div └── h1 // JavaScript access document.getElementById("app");
Note: The DOM only contains structure and content. It does not contain styling information.

Step 2: CSS Parsing -> CSSOM

CSS -> CSSOM

The browser downloads and parses CSS files. It converts all CSS rules into another tree called the CSS Object Model (CSSOM). Unlike the DOM, the CSSOM contains styling information only.

/* CSS Example */ h1 { color: blue; font-size: 24px; } // CSSOM stores: - Selectors - Specificity - Inheritance rules - Computed styles

Step 3: JavaScript Execution

JavaScript Modifies DOM & CSSOM

JavaScript can modify both the DOM and CSSOM. Because JavaScript can change HTML and CSS, the browser often pauses rendering while executing synchronous JavaScript. This is why large JavaScript files can delay page rendering.

// JavaScript can change content document.querySelector("h1").textContent = "New Text"; // This can trigger re-rendering

Step 4: DOM + CSSOM -> Render Tree

Render Tree Construction

The browser combines the DOM and CSSOM to create the Render Tree. The Render Tree contains only elements that need visual rendering.

<!-- Example --> <div>Hello</div> <div style="display:none">Hidden</div> // Second div is excluded from Render Tree // Important: - display: none -> Removed from Render Tree - visibility: hidden -> Present but invisible

Step 5: Layout (Reflow)

Calculate Positions

Once the Render Tree is ready, the browser calculates width, height, position, margins, padding, and borders for every visible element. This process is called Layout or Reflow.

/* Layout Calculation Example */ .container { width: 500px; } .box { width: 50%; } // Browser calculates: 50% of 500px = 250px // Properties that trigger Layout: width, height, margin, padding, font-size, display, position
Note: Layout is expensive because changing one element may require recalculating many others.

Step 6: Paint

Fill Pixels

After positions are known, the browser paints visual details including text, colors, borders, shadows, images, gradients, and backgrounds. Paint converts instructions into drawing commands. Layout already decided where elements go.

/* Paint draws these visual properties */ .box { background: blue; border: 2px solid black; box-shadow: 2px 2px 5px gray; } // Paint includes: - Text, Colors, Borders - Shadows, Images - Gradients, Backgrounds

Step 7: Compositing

Layer Assembly

Modern browsers split the page into multiple layers (fixed elements, transform animations, opacity animations, video, canvas). Each layer is painted separately, then the browser combines all layers into the final image. This is called Compositing.

GPU Acceleration

Hardware Acceleration

Certain CSS properties can be handled almost entirely by the GPU. The best examples are transform and opacity. These typically skip Layout and Paint, only requiring Compositing, making animations much smoother.

Fastest - Only Composite

transform, opacity

Medium - Paint + Composite

background-color, color, box-shadow

Slow - Layout + Paint + Composite

width, height, margin, padding, top, left, font-size, display

React & Rendering Pipeline

What Happens When React Updates UI?

React updates the Virtual DOM first, then compares old vs new Virtual DOM (Reconciliation). React then updates only the necessary real DOM nodes. After DOM changes, the browser runs parts of the rendering pipeline again.

DOM Change
+
Render Tree
+
Layout
+
Paint
+
Composite
Note: The browser does not necessarily repeat the entire pipeline from scratch. It performs only the minimum work required.

Final Summary

10 Key Points

1. Browser parses HTML -> DOM
2. Browser parses CSS -> CSSOM
3. JavaScript executes and may modify DOM/CSSOM
4. DOM + CSSOM -> Render Tree
5. Layout calculates size and position
6. Paint draws pixels
7. Composite combines layers
8. GPU helps composite layers efficiently
9. React minimizes DOM changes, reducing rendering work
10. transform and opacity are most performant for animations
